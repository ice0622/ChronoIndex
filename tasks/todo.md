## 現在の課題：チャプター精度が期待値に届いていない

### 現状の出力（14分の対談動画）
```
0:00  メンバー紹介
1:24  開発職の魅力
12:29  挑戦のススメ
```

### 期待値の出力（同じ動画）
```
0:00  メンバー紹介（河本・上田・沢田・楠）
1:24  入社動機① ものづくりへの情熱（沢田さん）
1:58  入社動機② 未経験からの挑戦（上田さん）
2:25  入社動機③ 待遇と成長環境（お二人）
3:00  エンジニアとして大切にしていること
...
12:29  参加者へのメッセージ
```

### ギャップの原因
現在の境界検出は「無音区間」と「定型フレーズ（ありがとうございました等）」だけを見ている。
対談動画では話題が流れるように変わるため、無音がなくてもトピックが変わる。

---

## 解決アプローチの選択肢

### 案A: 現在のアーキテクチャを改善（閾値チューニング）
- 無音検出の閾値を下げる（現在3〜30秒 → 2秒以上で区切る）
- **メリット**: 実装コスト低い
- **デメリット**: チャプター数が増えすぎる・話題と一致しない

### 案B: セマンティック境界検出（埋め込みベクトル）
- 連続するセグメントのテキストをベクトル化して類似度を計算
- 類似度が急落した点 = 話題転換として検出
- **ライブラリ**: sentence-transformers（無料・ローカル）
- **コスト**: 無料、初回モデルDL約500MB
- **精度**: 対談動画に効果的

### 案C: LLMに全文を渡して直接チャプター生成（最短）★推奨
- 文字起こし全文 + タイムスタンプをLLMに渡し「チャプターに分けて」と依頼
- 現在の「境界検出→タイトル生成」を「一括チャプター生成」に置き換え
- **コスト**: Gemini無料枠で1回のAPIコール
- **精度**: LLMがトピック理解をするため最も高い
- **速度**: 5〜10秒

## 問題の本質（2026-04-02 更新）

音声→テキスト変換で**話者情報がごっそり落ちている**のが根本原因。

```
現状のテキスト（誰が言ったか不明）:
  1:24  私は子供の頃から車とかバイクとか好きだったので...
  1:58  私はエンジニア未経験で転職先を探していましたので...

話者情報ありのテキスト（LLMがチャプター化できる）:
  1:24  [沢田] 私は子供の頃から車とかバイクとか好きだったので...
  1:58  [上田] 私はエンジニア未経験で転職先を探していましたので...
```

話者ラベルがあれば「沢田さんの発言をまとめたチャプター」が生成できる。

### Phase 5: 映像からの局所的テロップ抽出

**基本方針: 全フレームは処理しない**

無音・場面転換タイミングのフレーム1枚だけをOCRする。
境界検出で既にシーン切れ目を取得済みなので、**そのタイミングのスクショ1枚**だけが対象。

```
168分動画の場合：
  全フレーム処理: 168分 × 60秒 × 1fps = 10,080フレーム  ← 重い
  局所処理:       境界数 × 前後3フレーム ≒ 50〜150フレーム ← 軽い
```

**取得できる情報**:

| 情報 | 映像上の場所 | 処理方法 | コスト |
|------|------------|---------|--------|
| テロップ（名前・肩書き） | 画面下部 | OCR（Tesseract・無料） | 低 |
| スライドタイトル | 中央〜上部 | OCR | 低 |
| スライド本文 | 中央 | OCR | 低 |
| 顔認識・誰が映っているか | 顔領域 | Vision API（有料） | 高 |

**推奨アプローチ（コスト最小）**:
1. ffmpegで境界タイムスタンプのフレームを静止画として抽出
2. 画面下部（テロップ領域）だけクロップ
3. Tesseract OCRでテキスト抽出
4. 抽出テキストをそのタイムスタンプのセグメントに付加

### Phase 6: LLM全文チャプター生成

話者ラベル + テロップ情報付きの文字起こしをLLMに渡してチャプター生成。

```
入力イメージ:
  0:02  [SPEAKER_00] IT系インフラエンジニアの河本浩二です。
        [テロップ] 河本 浩二 / ITシステムソリューション事業部
  1:24  [SPEAKER_01] 私は子供の頃から車とかバイクとか好きだったので...
        [テロップ] 沢田 浩二 / ものづくり事業部
```

このインプットでLLMが生成できるチャプター:
```
0:00  メンバー紹介（河本・上田・沢田・楠）
1:24  沢田さん：ものづくりへの情熱と入社動機
1:58  上田さん：未経験から挑戦した理由
...
```

長尺動画のトークン問題:
- 168分 ≒ 約50,000トークン → Gemini 2.5 Flashの128Kコンテキストに収まる
- 問題なければ全文1回のAPIコールで処理
