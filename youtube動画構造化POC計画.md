# GENIAC-PRIZE 最終審査動画の構造化データ生成に関する包括的技術リサーチ報告書

## GENIAC-PRIZEの背景と長尺動画構造化の必要性

経済産業省と国立研究開発法人新エネルギー・産業技術総合開発機構（NEDO）が推進する「GENIAC-PRIZE」は、生成AIの社会実装を加速させるための画期的な懸賞金活用型プログラムである 。このプログラムは、製造業における暗黙知の形式知化、カスタマーサポートの生産性向上、官公庁の審査業務効率化、そして生成AIの安全性確保という、現代日本が直面する重要な課題に対して、具体的なAIソリューションを公募し、優れた成果に対して総額約8億円という巨額の懸賞金を授与するものである 。2026年3月24日に開催される最終審査・表彰式は、正午から19時40分までという、約8時間に及ぶ極めて長尺なオンライン配信を伴うイベントとして計画されている 。

このような長尺の動画コンテンツは、参加企業による最先端の技術プレゼンテーション、専門家による鋭い質疑応答、そしてAI戦略の第一人者である松尾豊教授やDeNAの南場智子氏といった著名な審査員による講評など、極めて密度の高い情報を含んでいる 。しかし、視聴者が特定の企業の発表や、特定の審査基準に関する議論を後から参照しようとする際、数時間に及ぶ動画の中から該当箇所を手動で探し出すことは極めて困難であり、情報のアクセシビリティが著しく低いという課題が存在する。YouTubeにおけるタイムスタンプやチャプター機能は、この課題を解決するための標準的な手法であるが、これを手動で作成するには膨大な工数が必要となる 。

本報告書では、GENIAC-PRIZEの最終審査動画を対象に、YouTubeのタイムスタンプに準拠した構造化データを自動生成するための技術的アプローチを検討する。イベントの構成テンプレート、司会の言語パターン、登壇者および審査員の動態、既存のチャプター成功事例、厳格な評価基準、そして音声認識（ASR）技術の限界という6つの観点から多角的に分析し、マルチモーダル大規模言語モデル（LLM）を核とした自動構造化手法と、その実効性を検証するための概念実証（POC）計画を詳述する。

| ------------- | -------------- | -------------------------------------- |
| 12:00 - 12:30 | オープニング         | 主催者挨拶（経済産業省・NEDO）、プログラム説明、審査員紹介        |
| 12:30 - 14:30 | テーマI：製造業（前編）   | ダイキン工業、三菱重工業、NanoFrontier等によるプレゼンテーション |

このスケジュールは、単なる時間の区切りではなく、情報のコンテクストが切り替わる重要な境界（バウンダリ）を示している。自動構造化システムは、これらのマクロな時間枠を事前知識として持ちつつ、実際の動画内での秒単位のズレを音声および映像信号から補正する必要がある。特に、2025年1月の最終選考会では、午前の部と午後の部に分かれた長時間のプログラムが組まれており、各セクションの開始時間が当初の予定から数分前後することが一般的である 。したがって、固定的なスケジュールに依存するのではなく、動的なイベント検知が求められる。

## 司会進行における言語パターンと遷移トリガー

動画のセグメント境界を特定する上で、司会者（MC）の発話は最も信頼性の高い「セマンティック・マーカー」として機能する。ピッチコンテストやシンポジウムにおける司会進行には、特定のフェーズ移行を示唆する定型的なフレーズ（言語パターン）が存在する 。

### 進行フェーズ別の典型的な言語表現

以下の表は、司会者が発話する可能性の高いフレーズと、それが示唆する構造化上の意味を整理したものである。

|**進行フェーズ**|**典型的なフレーズ（トリガー）**|**構造化における役割**|
|---|---|---|
|セグメント開始|「それでは、これより〇〇のセッションを開始いたします。」|新しいチャプターの開始点（Start Timestamp）|
|登壇者紹介|「続きましてのご登壇者は、〇〇株式会社、△△様です。よろしくお願いいたします。」|チャプタータイトルに含まれるべき固有名詞の特定|
|プレゼン終了|「〇〇様、ありがとうございました。」|プレゼンセクションの終端と質疑への移行|
|質疑応答開始|「それでは、これより質疑応答の時間とさせていただきます。審査員の皆様、いかがでしょうか。」|サブセグメント（Q&A）の開始点|
|質疑応答終了|「お時間となりましたので、質疑応答を終了させていただきます。〇〇様、ありがとうございました。」|登壇者ユニットの終了|
|審査移行|「以上をもちまして、全てのプレゼンテーションが終了いたしました。これより審査に入ります。」|待機・特別コンテンツセクションへの移行|

これらのフレーズは、音声認識（ASR）の結果に対して正規表現やセマンティック・サーチを適用することで抽出可能である。しかし、司会者が「それでは……」や「続きまして……」といった接続詞を多用するため、単一の単語ではなく、後続する名詞（テーマ名や企業名）との組み合わせを考慮した高度な文脈解析が必要となる。また、ライブイベントでは司会者の即興的な発言や、登壇者の交代に伴う無音時間、マイクの受け渡し音などの非言語的な音響イベントも発生するため、これらを「進行のノイズ」として適切に処理しなければならない 。

## YouTubeチャプター仕様と構造化の成功事例

自動生成される構造化データは、YouTubeのプラットフォーム仕様を完全に満たしつつ、視聴者にとっての有用性を最大化するものでなければならない。

### YouTubeタイムスタンプの技術的制約

YouTubeのシークバー上にチャプターを表示させるためには、以下の5つの基本条件を厳守する必要がある 。

1. **ゼロ秒開始の原則:** 最初のタイムスタンプは必ず「00:00」（または「0:00」）から開始しなければならない 。
    
2. **最小チャプター数:** 少なくとも3つ以上のタイムスタンプを設定する必要がある 。
    
3. **最小間隔の確保:** 各チャプターの長さは10秒以上でなければならない 。
    
4. **昇順配置:** タイムスタンプは時間の早い順に並んでいなければならない 。
    
5. **記述形式:** 「半角数字での時間表記 + 半角スペース + タイトル」という形式が推奨され、全角文字の使用は避けるべきである 。
    

### 既存の技術イベント動画における「善意の構造化」事例

成功している技術系イベントの動画では、視聴者が内容を一目で把握できるよう、以下のような構造化が行われている 。

- **00:00 オープニング：開催主旨と審査基準の説明**
    
- **15:45 【製造業】ダイキン工業×Fairy Devices：熟練者の技をAIで継承する**
    
- **32:10 質疑応答：現場での導入コストと教育効果について**
    
- **45:20 【安全性】NTTドコモ：日本語特化型ガードレール「chakoshi」の開発**
    
- **1:10:30 基調講演：松尾豊教授「生成AIが変える日本の産業構造」**
    

このように、単に「発表1」とするのではなく、「テーマ名 + 企業名 + プロジェクトの核心」という3つの要素を組み合わせたタイトリングが、視聴者の満足度を高める鍵となる。自動構造化システムにおいては、音声認識されたプレゼンの導入部から、これらのキーワードをLLMで抽出・統合する処理が求められる。

## GENIAC-PRIZE 評価基準に基づく意味的構造化

GENIAC-PRIZEの自動構造化において、他の一般的なイベント動画と一線を画す点は、動画の内容が「厳格な評価項目」に直結している点である。審査プロセスにおける議論を、これらの評価項目に基づいてタグ付けすることは、教育的・産業的な価値を付加することに繋がる。

### 審査項目と配点の構造

最終審査では、最大650点という配点の中で、情報探索性能、一致箇所表示、判断根拠の出力、ユーザビリティといった複数の観点からプロトタイプが評価される 。

| **評価項目** | **配点・重み**  | **構造化におけるタグ付けの意義**     |
| -------- | ---------- | ---------------------- |
| 情報探索の性能  | 最大300点（3倍） | AIの検索精度に関する議論の集約       |
| 一致箇所表示   | 最大200点（2倍） | UI上の証拠提示能力に関する議論       |
| 判断根拠の出力  | 最大100点（1倍） | LLMの論理的な説明性に関する議論      |
| ユーザビリティ  | 最大50点（1倍）  | 実際の業務フローへの適合性に関する議論    |
| 技術的優位性   | 加点対象       | 国産基盤モデルの活用、アーキテクチャの革新性 |

これらの評価項目は、審査員が質疑応答で投げかける質問の「意図」そのものである。たとえば、大塚玲氏が「このモデルのハルシネーション対策はどうなっていますか？」と質問した場合、それは「判断根拠の出力性能」や「安全性」というタグに関連付けられるべきである。構造化データ生成プロセスにおいて、LLMに対してこれらの評価項目をプロンプトとして与えることで、単なる時間的な分割を超えた、多角的な「意味インデックス」を生成することが可能となる。また、懸賞金の足切りライン（60%の390点以上）や、各順位の賞金額（1位1億円、2位7,000万円等）といったコンテクストも、結果発表セクションの要約において極めて重要な情報となる 。

## 音声認識の弱点と技術的制約の克服

自動構造化の基盤となる音声認識（ASR）技術には、現状いくつかの顕著な弱点が存在し、それらが構造化データの正確性を損なう要因となっている。これらを特定し、適切な対策を講じることがPOC計画の成功に不可欠である。

### ASRにおける主要な技術的課題

音声認識における主な失敗パターンと、それに対するエンジニアリング的な解決策を以下の表にまとめる。

| **弱点項目** | **発生原因と影響**                       | **構造化における対策**                             |
| -------- | --------------------------------- | ----------------------------------------- |
| 複数人の同時発話 | 質疑応答や交代時における声の重なり。認識漏れや混合が発生する。   | 話者分離（Diarization）に強いWhisperX等の採用 。        |
| 専門用語の誤変換 | 「ジェイルブレイク」「IP」「LLM」等のIT用語の誤認識。    | 専門用語辞書の事前登録およびLLMによる後補正 。                 |
| 背景ノイズと反響 | 会場内の拍手、BGM、マイクの反響。精度が著しく低下する。     | 信号処理（デノイズ）の適用と、高音質な配信用音声の使用 。             |
| 代名詞と文脈欠如 | 「それは」「こちらの」等の指示語。単独の要約では意味が不明になる。 | 前後の発話およびスライドOCR情報を用いた照応解析。                |
| 長時間の蓄積誤差 | 8時間の動画を一度に処理する際の処理負荷とメモリ制限。       | 1時間程度のウィンドウによる反復的な予測処理（Chapter-Llama方式） 。 |

特に、GENIAC-PRIZEのような技術イベントでは、登壇企業が「国産AI『PLaMo』」や「Tig溶接技術」といった極めて固有かつ重要な固有名詞を多用する 。これらが「プラモ」や「キグ溶接」のように誤認識されると、チャプタータイトルの信頼性が失われる。これを解決するためには、音声から得られたテキストをそのまま使うのではなく、NEDOが公開しているプレスリリースや最終候補者リスト、および提案書の要約などの外部知識をLLMにRAG（検索拡張生成）の形式で与え、正確な名称に修正するプロセスが必須となる 。

## 自動構造化手法：マルチモーダルLLMパイプラインの検討

最新の研究動向を反映した、GENIAC-PRIZE動画の自動構造化のための技術アーキテクチャを提案する。このアプローチは、単一のモダリティに依存するのではなく、音声、映像（フレーム）、およびテキスト（OCR・外部知識）を統合した「マルチモーダル・セマンティック・インデクシング」を中核とする 。

### 推奨される技術スタックと処理フロー

自動構造化システムは、以下のパイプラインを通じて実行される。

1. **音声解析層（Audio Intelligence）:**
    
    - **モデル:** WhisperX または OpenAI Whisper (large-v3)。
        
    - **処理:** 8時間の音声から文字起こしを行い、タイムスタンプ付きのセグメントを生成する。同時に、話者ダイアライゼーションを行い、MC、登壇者、審査員の各発話をラベル付けする 。
        
2. **映像解析層（Visual Intelligence）:**
    
    - **手法:** PySceneDetect を用いたショット境界検出 。
        
    - **OCR:** キーフレームからスライド内のタイトルや箇条書きテキストを抽出する。特に、スライドの表紙（企業名表示）や「まとめ」スライドの検知は、セグメントの区切りを特定する強力な視覚的ヒントとなる 。
        
3. **セマンティック統合層（LLM Reasoning）:**
    
    - **モデル:** Chapter-Llama または Google Gemini 1.5 Pro（100万トークンのコンテクストウィンドウを活用） 。
        
    - **入力:** ASRテキスト、話者情報、OCRテキスト、およびGENIAC-PRIZE公式プログラム。
        
    - **タスク:** 各モダリティの矛盾を解消し、セグメントの境界 $t$ を予測するとともに、その区間にふさわしい「YouTube形式のタイトル」を生成する 。
        
4. **構造化出力層（Output Standardization）:**
    
    - YouTubeの規約（00:00開始、昇順、10秒間隔）に適合するよう、最終的なタイムスタンプをバリデート・修正する 。
        

### 情報抽出の精度を高める「Speech-guided Frame Selection」

長尺動画の全フレームを解析するのは計算コストが高すぎるため、音声の文字起こし内容をガイドとして重要なフレーム（スライドの切り替わり等）を選択的に抽出し、LLMに供給する手法（Speech-guided frame selection）が有効である 。これにより、1時間以上の動画であっても、単一のフォワードパスで一貫性のあるチャプター生成が可能となる。研究事例によれば、この手法を用いたLLMベースのアプローチは、従来の手法と比較してF1スコアで45.3% vs 26.7%という大幅な精度向上を達成している 。

## POC（概念実証）計画の策定

提案した自動構造化手法の実効性を検証し、2026年3月の本番運用に備えるためのPOC計画を、以下の3つのフェーズで実施する。

### フェーズ1：データの収集とベースラインの構築（期間：4週間）

このフェーズでは、過去のアーカイブ動画を用いて、技術的な土台を固める。

- **使用データ:** 2025年1月および2024年12月のGENIAC-PRIZE最終選考会・表彰式動画 。
    
- **実施内容:**
    
    - 既存のASRモデル（Whisper）単体での精度測定。
        
    - 手動で作成された「正解タイムスタンプ」とのズレ（Error Seconds）の定量化。
        
    - 専門用語辞書の構築と、NEDO提供ドキュメントからのキーワード抽出 。
        
- **成功指標:** 単純ASRでのキーワード認識率が80%を超えること。
